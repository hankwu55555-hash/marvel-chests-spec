## 3.1 流程圖

```mermaid
flowchart TD
    A([玩家旋轉轉輪]) --> B{是否達成\nEpic Win？}
    B -- 否 --> Z([流程結束])
    B -- 是 --> C{觸發來源？}

    C -- Instant Bonus\nWild Bisons --> D[不計入 Marvel Chests]
    D --> D1[UI 顯示\n「本次不計入」提示]
    D1 --> Z

    C -- 一般轉輪 --> E{寶箱欄位\n是否已滿？\n4/4}
    E -- 是 --> F[不發放寶箱]
    F --> F1[UI 顯示\n「欄位已滿」Toast]
    F1 --> Z

    E -- 否 --> G[依下注區間\n計算稀有度權重]
    G --> H[發放對應等級寶箱\n放入玩家欄位]
    H --> I[玩家進入寶箱選單]
    I --> J{是否啟動計時？\n點擊 START}

    J -- 否 --> K([寶箱待命中\n等待手動啟動])

    J -- 是 --> L{是否有其他寶箱\n正在計時？}
    L -- 是 --> M([排隊等待\n前一個完成後\n手動啟動])
    L -- 否 --> N[開始倒數計時\n伺服器端執行]

    N --> O{玩家選擇\n開箱方式？}
    O -- 等待倒數完成 --> P[倒數歸零]
    O -- 寶石即時開箱 --> Q[扣除寶石\n依剩餘時間計費]

    Q --> R{寶石是否足夠？}
    R -- 否 --> S[顯示「寶石不足」\n導向購買頁]
    S --> O
    R -- 是 --> T[跳過等待]

    P --> U[寶箱狀態 → 已完成 ✅]
    T --> U
    U --> V[玩家點擊 COLLECT]
    V --> W[播放開箱動畫\n依稀有度差異化]
    W --> X[發放獎勵\n幣值 + May Win]
    X --> Y[釋放欄位\n可接收新寶箱]
    Y --> Z
```

## 3.2 流程階段說明

### 階段一：觸發判定

| 步驟 | 說明 |
|------|------|
| 轉輪旋轉 | 玩家執行一般 Spin |
| Epic Win 判定 | 系統判定本次是否達成 Epic Win |
| 來源檢查 | 排除 Instant Bonus / Wild Bisons 觸發的 Epic Win |
| 欄位檢查 | 確認玩家寶箱欄位未滿（< 4 個） |

### 階段二：寶箱發放

| 步驟 | 說明 |
|------|------|
| 稀有度計算 | 依當次 Total Bet 所在下注區間，計算各稀有度機率權重 |
| 寶箱入袋 | 發放對應等級寶箱至玩家欄位 |

### 階段三：計時與開箱

| 步驟 | 說明 |
|------|------|
| 啟動計時 | 玩家手動點擊 START，開始伺服器端倒數 |
| 單一計時限制 | 同一時間僅允許 1 個寶箱倒數 |
| 開箱選擇 | 等待倒數完成（免費）或使用寶石即時開啟（付費） |
| 寶石費用 | 依剩餘時間計算，最低 1 顆寶石 |

### 階段四：獎勵發放

| 步驟 | 說明 |
|------|------|
| 收取操作 | 倒數完成後玩家點擊 COLLECT |
| 開箱動畫 | 依稀有度播放對應動畫（詳見 [05-演出動態](05-animations.md)） |
| 獎勵結算 | 發放保證幣值 + 機率性 May Win 額外獎勵 |
| 欄位釋放 | 該寶箱位置清空，可接收新寶箱 |

---

## 3.3 關鍵分支彙整

| 分支情境 | 系統行為 | UI 回饋 |
|---------|---------|---------|
| 非 Epic Win | 流程不進入寶箱系統 | 無 |
| Instant Bonus / Wild Bisons 觸發 | 不發放寶箱 | 顯示「本次不計入 Marvel Chests」 |
| 欄位已滿（4/4） | 不發放寶箱 | Toast：「欄位已滿，請先開啟現有寶箱」 |
| 已有寶箱正在計時 | 新寶箱無法啟動計時 | 排隊等待，需手動啟動 |
| 寶石不足 | 無法即時開箱 | 顯示「寶石不足」+ 導向購買頁 |
| App 關閉 / 斷線 | 伺服器端計時持續 | 重開後顯示正確剩餘時間或已完成 |
| 倒數完成未收取 | 寶箱保持已完成狀態 | 顯示 COLLECT 按鈕，不過期 |
