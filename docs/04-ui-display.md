## 4.1 寶箱選單主面板

玩家點擊入口後展開全螢幕面板，標題為「MARVEL CHESTS」。

| UI 元素 | 說明 |
|---------|------|
| 標題列 | 「MARVEL CHESTS」大字標題，金色描邊 |
| ⓘ 按鈕 | 左上角，點擊進入 How to Play 說明頁 |
| ✕ 關閉按鈕 | 右上角，關閉面板回到主遊戲畫面 |
| 寶箱欄位 | 共 4 格橫向排列，每格為獨立卡片 |

### 單格卡片內容

每個寶箱欄位卡片由上至下包含：

| 區塊 | 說明 |
|------|------|
| P 點徽章 | 寶箱左上角圓形徽章，顯示開箱可獲得的 P 點數（Chest Odyssey 活動開啟時） |
| 寶箱圖示 | 依稀有度顯示對應外觀（Common 橘棕木箱 / Rare 銀藍金屬箱） |
| 狀態標籤 | 未啟動顯示等待時間（如「2HRS」「60MIN」）；倒數完成顯示「READY!」 |
| 稀有度標籤 | 文字標示「COMMON」「RARE」「EPIC」「LEGENDARY」 |
| WIN AT LEAST | 顯示保證最低幣值獎勵金額（幣值圖示 + 數字） |
| MAY WIN 區域 | 以小圖示列出可能額外獎勵品項（寶石、Ball Shots 等） |
| 操作按鈕 | 依狀態顯示不同按鈕（見下方 §4.2） |

### 空欄位

- 未持有寶箱的欄位顯示**虛線輪廓**
- 欄位內文字提示：「HIT AN EPIC WIN TO GET A CHEST」
- 引導玩家回到轉輪遊玩以獲得新寶箱

## 4.2 操作按鈕狀態

| 寶箱狀態 | 按鈕 | 樣式 | 行為 |
|---------|------|------|------|
| 未啟動計時 | **START!** | 橘色大按鈕 | 點擊啟動倒數計時 |
| 計時進行中 | **START!**（排隊中） | 橘色大按鈕，但已有其他箱在計時時無法啟動 | 提示需等待前一個完成 |
| 倒數完成 | **COLLECT** | 綠色大按鈕 | 點擊進入開箱動畫 |
| 計時中（寶石即開） | **💎 即時開啟** | 計時器旁顯示寶石費用 | 扣除寶石跳過等待 |

## 4.3 押注段寶箱資格提示（ELIGIBLE FOR）

玩家調整 TOTAL BET 時，UI 即時顯示當前下注區間可獲得的寶箱等級。

| UI 元素 | 說明 |
|---------|------|
| 標題 | 「ELIGIBLE FOR」 |
| 寶箱圖示列 | 橫向排列 4 個寶箱圖示（Common → Rare → Epic → Legendary） |
| 解鎖狀態 | 已解鎖等級：圖示亮起 / 勾選；未解鎖等級：圖示暗化 + 鎖頭圖示 |
| 氣泡提示 | 未解鎖的最近等級顯示「unlock at the next level」提示氣泡 |
| 進度條 | 寶箱圖示下方，顯示當前下注距離解鎖下一等級的進度 |

### 顯示時機

- 玩家點擊 TOTAL BET 旁的 ＋/－ 按鈕調整押注金額時即時更新
- 隨下注金額增加，寶箱圖示逐步從左至右解鎖
- 讓玩家直觀理解「提高下注 → 解鎖更高稀有度寶箱」的關係

### 設計目的

- 降低資訊落差：玩家不需查閱說明即可理解下注與稀有度的關係
- 刺激加注意願：視覺化的解鎖進度激勵玩家提高下注金額

## 4.4 迷你列（轉輪上方快捷列）

寶箱選單在主遊戲畫面上方提供一個**迷你橫向列**，讓玩家無需開啟全螢幕面板即可快速操作。

頂部橫幅文字：「SPIN & HIT AN EPIC WIN TO WIN CHESTS」

| UI 元素 | 說明 |
|---------|------|
| 寶箱縮圖 | 4 格橫向小型寶箱圖示，依稀有度顯示對應外觀 |
| P 點徽章 | 同主面板，顯示各寶箱 P 點數（Chest Odyssey 活動開啟時） |
| 狀態標籤 | 每個寶箱下方顯示當前狀態（見下方輪播規則） |
| 展開按鈕 | 右上角 ↗️ 圖示，點擊展開至全螢幕 Marvel Chests 主面板 |
| 進度條區域 | 下方為 Chest Odyssey 通行證進度（Chest Odyssey 活動開啟時） |

### 狀態標籤輪播規則

計時中的寶箱，其狀態標籤會在**倒數時間**與**寶石即開費用**之間輪播切換：

| 輪播幀 | 顯示內容 | 範例 |
|--------|---------|------|
| 幀 A | 剩餘倒數時間（HH:MM:SS） | `01:52:28` |
| 幀 B | 寶石即開費用（💎 + 數量） | `💎120` |

- 兩幀交替顯示，讓玩家同時掌握「還要等多久」和「花多少寶石可以跳過」
- 寶石費用隨剩餘時間減少而降低（如同一箱從 💎120 逐步降至 💎48）
- 未啟動計時的寶箱不輪播，僅顯示等待時間文字（如「2HRS」）
- 倒數完成的寶箱顯示 COLLECT 按鈕，不輪播

### 各欄位狀態對照

| 寶箱狀態 | 迷你列顯示 |
|---------|-----------|
| 空欄位 | 空底座（粉紅 / 紫色圓台） |
| 未啟動計時 | 寶箱靜態 + 等待時間文字 |
| 計時中 | 寶箱 + 輪播標籤（時間 ↔ 寶石費用） |
| 倒數完成 | 寶箱 + COLLECT 按鈕 |

玩家點擊 COLLECT 後，進入開箱預覽 / 結果畫面：

| UI 元素 | 說明 |
|---------|------|
| 寶箱外觀 | 依稀有度顯示對應材質與色調 |
| WIN AT LEAST | 顯示保證最低幣值獎勵金額 |
| MAY WIN 區域 | 以圖示列出可能額外獎勵品項 |

## 4.6 開箱結果畫面

開箱動畫播放完畢後顯示結果畫面：

| UI 元素 | 說明 |
|---------|------|
| 標題 | 「[稀有度] CHEST」，如「COMMON CHEST」 |
| 寶箱圖示 | 已開啟狀態，蓋子掀開 |
| 背景特效 | 金色放射光芒（依稀有度強弱不同） |
| 幣值獎勵 | 金幣圖示 + 實際獲得金額（如「12B」） |
| P 點獎勵 | P 圖示 + 數量（如「×10」）（Chest Odyssey 活動開啟時） |
| 額外獎勵 | 若有 May Win 掉落，逐項以圖示 + 數量呈現 |
| COLLECT 按鈕 | 綠色大按鈕，確認收取所有獎勵，返回寶箱選單 |

## 4.7 Toast 通知規範

系統在特定情境下觸發前端 Toast 通知：

| 觸發情境 | Toast 內容 | 優先級 |
|---------|-----------|--------|
| 欄位已滿時觸發寶箱條件 | 「寶箱欄位已滿，請先開啟現有寶箱以接收新獎勵！」 | 高 |

| 寶石不足以即時開箱 | 「寶石不足」+ 導向寶石購買頁入口 | 中 |
| 倒數完成（若玩家在線） | 「你的 [等級] 寶箱已可開啟！」 | 低 |

點擊主面板左上角 ⓘ 按鈕進入說明頁：

| 頁面 | 內容 |

| 第 3 頁 | 寶石即時開箱說明 |

## 4.9 Chest Odyssey 活動元素

以下元素僅在 Chest Odyssey 通行證活動開啟時顯示，非 Marvel Chests 常駐功能：

| 元素 | 說明 |
|------|------|
| P 點徽章 | 各寶箱左上角顯示可獲得的 P 點數（Common = 10P、Rare = 20P） |
| P 點獎勵 | 開箱結果畫面中額外顯示 P 點獲得數量 |
| Chest Odyssey 通行證 | 迷你列下方的進度條，累積 P 點解鎖里程碑獎勵 |
| 活動倒數計時 | 顯示 Chest Odyssey 活動剩餘時間（如「1d 21h」） |
| Marvel Chests 入口按鈕 | 主畫面右下角「Event! Marvel Chests」按鈕（Chest Odyssey 活動期間樣式可能強化） |

### Chest Odyssey 通行證

- 開箱獲得 P 點，累積至里程碑門檻可領取額外獎勵
- 進度條上標示各里程碑所需 P 點（如 10 → 30 → 50 → 80 …）
- 各里程碑獎勵為幣值（如 810M、1.6B 等）或道具

> ⚠️ **待確認**：Chest Odyssey 通行證的里程碑數值與獎勵內容，需由數值企劃另行提供。
