# 05 · 邊界情境（Edge Cases）

[← 回目錄](../README.md)

---

| # | 情境 | 預期行為 |
|---|------|---------|
| 1 | 寶箱欄位已滿（4/4），玩家再次觸發 Epic Win | 系統不發放新寶箱；前端顯示 Toast 提示「欄位已滿，請先開啟現有寶箱」 |
| 2 | Instant Bonus 或 Wild Bisons 觸發 Epic Win | 不計入 Marvel Chests 資格，不發放寶箱；觸發當下 UI 即時標示「本次不計入 Marvel Chests」 |
| 3 | 玩家在計時中途關閉 App 或斷線 | 計時在伺服器端繼續；重新開啟 App 後顯示正確剩餘時間或「已完成」狀態 |
| 4 | 倒數完成但玩家未即時回來收取 | 寶箱保持「已完成 ✅」狀態等待收取；不自動開啟、不過期 |
| 5 | 玩家嘗試用寶石開啟剩餘秒數極少的箱子 | 最低消費門檻為 1 顆寶石；即使剩餘 < 1 分鐘仍需付費，不可免費跳過 |
| 6 | 玩家寶石不足以即時開箱 | 顯示「寶石不足」提示，並提供入口導至寶石購買頁 |

## 補充說明

### Edge Case 2：限制條件 UI 強化

目前遊戲說明僅在 How to Play 第 2 頁提及此限制，玩家容易忽略。建議：

1. 在 How to Play 頁面以**紅色警示框**強調
2. 在 Instant Bonus / Wild Bisons 的 Epic Win 結算畫面上加入**即時標示**
3. 客服 FAQ 補充相關說明

### Edge Case 1：滿格補償設計（優化建議）

滿格時的 Epic Win 直接消失，容易讓玩家感到「浪費」。建議研究：

- 溢出箱暫存機制
- 或轉換為少量幣值/其他補償

詳見 [優化建議](06-improvements.md#2-寶箱滿格的補償設計)
